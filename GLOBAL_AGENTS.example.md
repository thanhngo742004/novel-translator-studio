# Global Codex Instructions Example

Copy this to `~/.codex/AGENTS.md` if you want global behavior.

- Always inspect project instructions before editing files.
- Prefer planning before implementation for large projects.
- Ask before adding production dependencies.
- Never commit secrets, API keys, tokens, or private data.
- Use small, reviewable changes.
- When modifying code, run relevant tests or explain why tests cannot run.
- For architecture-heavy tasks, produce docs/plans before scaffolding code.
